## 1) Картина рынка: три зоны
- **Красная зона** — «мёртвая» монета: боковик на низких объёмах. **Не торгуем.**
- **Зелёная зона** — актив *в игре*: высокий относительный объём (HRV) и тренд.
- **Синяя зона** — волатильный боковик *после пампа*. Торгуем при выполнении контекста по количеству сделок.

## 2) Фильтр допуска: три контекста
Проверяются за последние 24 часа **на фьючерсах** (спот не учитывать).
1) **≥ BTC по количеству сделок** — допускается агрессивный стиль: идеи, первые импульсы, скальпинг; можно смягчать жёсткие требования к «числу касаний» в формациях.
2) **Высокий относительный объём (HRV)** — актив *в игре*: допускается активная работа по тренду, возможны послабления (детали — в модуле про потенциалы).
3) **≥ 0.5× BTC по количеству сделок** — *строгий режим*: работаем **только** по правилам формаций/стопов/потенциалов, без послаблений.

**Исключения**  
- **BTC и ETH** — всегда допускаются к торговле (но с соблюдением правил стратегии).
- **Листинг < 2 недель** — можно рассматривать как «в игре» даже без порога по сделкам (контроль рисков).
- **Недавний памп на D1** — монету можно рассматривать (строгий режим), даже если текущие метрики контекста не дотягивают.

## 3) Правила по зонам
- **Красная** — запрет на входы (ожидание признаков «в игре»).
- **Зелёная (HRV)** — работать по тренду; возможны входы на первых импульсах и идеи. Послабления допустимы **при контроле риска**.
- **Синяя (волатильный боковик)**  
  - если **≥ BTC** по сделкам — можно «агрессивно»: первые импульсы, скальпинг, пробои наклонных без долгого ожидания касаний;  
  - если **0.5×–1× BTC** — *строго*: торгуем **только** по полным правилам (ожидание нужного числа касаний и т.п.).

## 4) Таймфрейм — как выбирать
- Базовый якорь — **D1** (от него вниз: H4/H1 → M15/M5 → M1).
- Таймфрейм **динамичен**: HRV может проявиться на любом масштабе — от M1 до H1/Н30 и выше.
- Суть не в «любимом ТФ», а в том, **где проявился HRV и читается структура** (формации/зоны).
- На неликвидах «игра» может быть видна уже на M1–M5; на крупных активах — на Н30–H1–D1.

## 5) Дерево решения (допуск и стиль)
1) Инструмент = BTC/ETH? → **Да** → допускается (соблюдать правила). **Нет** → п.2.  
2) Листинг < 2 недель или на D1 был недавний памп? → **Да** → допускается (строгий режим). **Нет** → п.3.  
3) Сделки ≥ BTC за 24ч (фьючерсы)? → **Да** → «агрессивный» режим. **Нет** → п.4.  
4) HRV присутствует? → **Да** → «актив в игре» (работа по тренду). **Нет** → п.5.  
5) Сделки ≥ 0.5× BTC? → **Да** → «строгий» режим. **Нет** → **пропуск** (красная зона/нет контекста).

## 6) Чек-листы
**A. Допуск к торговле (быстрый фильтр)**  
- [ ] Это фьючерсные сделки за 24 часа, а не спот.  
- [ ] ≥ BTC **или** HRV **или** ≥ 0.5× BTC (строгий режим).  
- [ ] Для «исключений» (BTC/ETH, листинг, недавний памп) — описан риск и режим.  

**B. Работа в зелёной зоне (HRV)**  
- [ ] Тренд подтверждён; структура читается.  
- [ ] Входы по правилам стратегии; допустимы идеи/первые импульсы.  
- [ ] Риски: фиксирован SL, не раздувать размер позиции.

**C. Работа в синей зоне (волатильный боковик)**  
- Режим **≥ BTC**: допускаются пробои наклонных без ожидания 3–4 касаний; скальпинг первых импульсов.  
- Режим **0.5×–1× BTC**: строго ждать формальные критерии формаций, план SL/TP до входа.

**D. Выбор таймфрейма**  
- [ ] D1 → выделены зоны/контекст.  
- [ ] Ниже — тот ТФ, **где виден HRV и «рисунок»**.  
- [ ] ЛТФ для триггера (лом структуры/SR‑flip), МТФ для цели.

## 7) Частые ошибки
- Жёсткая «привязка» к одному ТФ.  
- Попытка торговли в красной зоне.  
- Учёт **спот**‑сделок вместо **фьючерсов**.  
- Игнор HRV в пользу «любимого» масштаба.  
- Неверное толкование порога «в два раза меньше» (это **порог ≥ 0.5× BTC**, а не «что угодно ниже»).

## 8) Мини‑SOP — от скринера до входа
1) Фьючерсы: собери метрику «сделки за 24ч» по альтам и BTC.  
2) Отметь **зелёные** (HRV) и **синие** зоны на D1/НТF.  
3) Присвой режим («агрессивный» / «строгий» / «пропуск»).  
4) Выбери ТФ, где структура **читается**, найди триггер.  
5) Пропиши SL/TP/размер до клика; журналируй скрин «до/после».
