> База/баланс формируется там, где покупатели ≈ продавцам; импульс/дисбаланс — там, где одна сторона доминирует. Зона спроса — там, где в прошлом было доминирование покупателей; зона предложения — доминирование продавцов. Зоны **слабеют с каждым касанием**, а большие «балансы» цена чаще **прошивает**, реагируя на бывшие «импульсы».

## 1) Базовые определения (коротко)
- **Баланс (проторговка)** — компактный боковик после/между импульсами («база»).
- **Дисбаланс (импульс)** — резкий направленный участок (рост/снижение).
- **Зона спроса (Demand)** — там, где после базы последовал **рост** (превышение спроса).
- **Зона предложения (Supply)** — там, где после базы последовало **снижение** (превышение предложения).

## 2) Четыре шаблона зон
(обозн. латиницей — привычные в торговых планах)
- **RBR — Rally → Base → Rally**: зона **спроса** (продолжение вверх).
- **DBR — Drop → Base → Rally**: зона **спроса** (разворот вверх).
- **RBD — Rally → Base → Drop**: зона **предложения** (разворот вниз).
- **DBD — Drop → Base → Drop**: зона **предложения** (продолжение вниз).

## 3) Как размечать зону (минимальные критерии)
1) Найди **импульс** → затем **базу** (не микроскопическую) → затем **второй импульс** в соответствии со схемой RBR/DBR/RBD/DBD.  
2) Границы зоны: по **крайним значениям базы** (консервативно — включая хвосты; агрессивно — по телам).  
3) **Контекст**: зона сильнее, если совпадает со старшим ТФ и при наличии всплеска относительного объёма.  
4) Свежесть: **первое возвращение** в зону — самое статистически сильное.

## 4) Правила силы/ослабления зоны
- Чем **больше касаний**, тем **слабее** зона (ликвидность «выедается»).  
- Цена часто **прошивает протяжённые «балансы»**, останавливаясь у «следующей» зоны дисбаланса.  
- Чем **компактнее** и «чище» база и чем **энергичнее** выход из неё, тем зона сильнее.  
- **Перекрытие** с зонами старшего ТФ усиливает приоритет; конфликт старшего ТФ — красный флаг.

## 5) Применение в сделке (скелет плана)
**Вход** (варианты-триггеры):
- Ретест края зоны с быстрым откликом (реакция свечой/кластером).  
- Младший ТФ: локальный **S/R-flip** или **лом структуры** внутри зоны.  
- Подтверждение объёмом/лентою — по системе трейдера.

**Стоп-лосс:**
- За **противоположной границей** базы (плюс буфер), либо по метрике волатильности (ATR).  
- Детали ролевого урока по стопам выносились в отдельный модуль — здесь задаём только каркас.

**Цели:**
- Ближайшая противоположная зона (Demand↔Supply).  
- Частичный выход в «балансе», который цена склонна прошивать на инерции.

## 6) Чек-листы
**A. Идентификация зоны (универсальный)**  
- [ ] Видны два импульса, разделённые базой.  
- [ ] База **смотрибельная** (не микробокс).  
- [ ] Границы по базе отчерчены, выбран режим (хвосты/тела).  
- [ ] Совпадение со старшим ТФ/уровнем усиливает сценарий.  
- [ ] Зона **свежая** (первое-второе касание).

**B. План входа из зоны спроса (RBR/DBR)**  
- [ ] Тренд/контекст не противоречит покупкам.  
- [ ] Возврат в зону — быстрый отклик/паттерн подтверждения.  
- [ ] Стоп-лосс за нижней границей базы (+буфер).  
- [ ] Цель — ближайшая зона предложения/узкий баланс.

**C. План входа из зоны предложения (RBD/DBD)**  
- [ ] Тренд/контекст не противоречит продажам/шортам.  
- [ ] Возврат в зону — отклик/лом ЛТФ структуры вниз.  
- [ ] Стоп-лосс за верхней границей базы (+буфер).  
- [ ] Цель — ближайшая зона спроса/узкий баланс.

## 7) Частые ошибки (и как их избежать)
- Торговля **внутри большого баланса** без привязки к зонам — ждать край зон/импульс.  
- Вход в **много раз тестированную** зону — приоритизировать свежие.  
- Разметка «базы» как **микродиапазона** — проверять масштаб относительно импульса.  
- Игнор контекста старшего ТФ — всегда сверху вниз: HTF → MTF → LTF.

## 8) Мини‑SOP (пошагово)
1) Выбери ТФ (например, D→H1→M5).  
2) На HTF найди ключевые зоны (RBR/DBR/RBD/DBD).  
3) На MTF отметь **свежесть** и приоритет (конфликты/совпадения).  
4) На LTF дождись **триггера**; заранее поставь SL/TP.  
5) В журнал — скрин до/после, оценка касаний и «качества» зоны.
