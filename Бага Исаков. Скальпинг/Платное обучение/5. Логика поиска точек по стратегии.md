> Важно: «логика» = фильтрация (контексты) → вочлист → проверка зон/уровней → поиск формации на рабочем ТФ → правила стопа/цели. Скальпинг вне рамок этой методички.

## 1) Контексты допуска (напоминание)
- **Контекст 1 (≥ BTC по количеству сделок, фьючерсы за 24ч)** — можно работать агрессивнее: первые импульсы, пробои наклонных с меньшим числом касаний.  
- **Контекст 2 (высокий относительный объём — HRV)** — «актив в игре»: работаем по тренду, допускаются расширенные цели.  
- **Контекст 3 (≥ 0.5× BTC по сделкам)** — строгий режим: только по формальным критериям формаций.

*Исключения*: недавний листинг, «близость к важному уровню на D1/ATH/ATL» — можно держать в вочлисте, но входы строго по правилам.

## 2) Ежедневный алгоритм (отбор → вочлист)
1) **Скринер по кол-ву сделок (фьючерсы, 24ч)**: отсеять активы с контекстом 1/2/3.  
2) **Проверить дневной график (D1)**: зоны спроса/предложения, уровни дня, близость к ATH/ATL.  
3) **Пометить «почти подходящие»**: где сделки «дотягиваются» до порога у уровня (ожидаем добора объёма при подходе к ключевому уровню).
4) **Завести вочлист**: монета, контекст(ы), причина (HRV / у уровня / листинг), рабочие ТФ для мониторинга.

## 3) От вочлиста к точке входа (шаги на МТФ/ЛТФ)
1) Перейти на **H1/M15/M5** (по ситуации) и искать *одну из разрешённых формаций*:  
   - пробой **уровня дня**,
   - **наклонка → пробой** (3-е подтверждение, затем пробой),
   - **проторговка после отката** по тренду,
   - **обратка** (когда ожидаемое продолжение сорвалось — аккуратно).
1) Проверить, что условия формации **соответствуют контексту** (например, для контекста 3 не «срезаем углы» по числу касаний).
2) На **рабочем ТФ** у цены должен быть триггер: поджатие, лом структуры/SR-flip, ретест края зоны.
3) Стоп **за ближайшим балансом**, цели — по модулю «Потенциалы» (методичка 04).

## 4) «Идейные» сделки — правила допуска
- Базово «идейная» допустима в **Контексте 1 (≥ BTC)**.
- При **только HRV (Контекст 2)** — допускается **только если**:
  - HRV виден на **D1** (не локальный всплеск),
  - рядом **ATH/ключевой уровень**,
  - есть «чистая» формация и баланс для стопа.
- На **Контексте 3** идейные **не запускать**.
Примечание: в лекции приводится пример «исключения по голове» — это опыт‑зависимая часть. В методичке фиксируем «жёсткий» вариант, чтобы не размывать дисциплину.

## 5) Редкость и дисциплина
- Качественных точек **мало**: от 1 до 5 сделок в неделю; далеко не каждая из «заметок» сработает в сделку.
- Сигналы публикуются редко и соответствуют личным входам автора — ориентир по качеству отбора.

## 6) Чек‑листы
**A. Отбор и вочлист**
- [ ] Контекст 1/2/3 подтверждён (фьючерсы, 24ч).
- [ ] На D1 отмечены зоны/уровни/ATH‑ATL.
- [ ] «Почти подходящие» добавлены (ожидаемый добор сделок у уровня).
- [ ] Для каждой монеты записаны рабочие ТФ и причины наблюдения.

**B. Поиск точки (МТФ→ЛТФ)**  
- [ ] Ищем одну из разрешённых формаций (уровень дня / наклонка / проторговка / обратка).
- [ ] Проверяем соответствие требованиям формации под текущий контекст.
- [ ] Есть триггер (поджатие, лом, ретест), стоп можно спрятать **за баланс**.
- [ ] Цели заданы по «Потенциалам»; Risk$ фиксирован.

**C. Идейная сделка (если есть)**
- [ ] Контекст 1 **или** HRV на **D1** + близко ATH/ключевой уровень.
- [ ] Формация «чистая», есть баланс для стопа.
- [ ] План частичных выходов и отмены сценария.

## 7) Частые ошибки
- Игнор «строгого режима» в Контексте 3 (срезание условий формаций).
- Вход без баланса для стопа (см. Методичка 04).
- Погоня за «каждым движением» — точек мало по определению.
- Ведение учёта причин вочлиста «на память», а не в таблице.

## 8) Мини‑SOP «от метрик к сделке»
1) Скринер сделок (фьючерсы, 24ч) → присвоить контекст.
2) D1: отметить зоны, уровни, ATH/ATL.
3) Сформировать вочлист (причина, контекст, ТФ).
4) МТФ/ЛТФ: искать формацию, получить триггер.
5) Стоп за баланс, цели по контексту (см. М‑04), Risk$ фиксирован.
6) Сделать запись в журнал: скрин до/после, причина входа, контекст, R:R.

## 9) Упражнения
- Сформируй вочлист из 10 монет с классификацией по контекстам и причинами наблюдения.
- Для 5 кейсов «почти подходит по сделкам» распиши сценарий добора объёма у уровня и какие триггеры будешь ждать.
- Собери 6 примеров входов: 2 — по уровню дня, 2 — по наклонке, 1 — по «обратке», 1 — «проторговка после отката».
